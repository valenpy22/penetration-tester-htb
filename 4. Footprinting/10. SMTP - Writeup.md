# SMTP Enumeration — HTB Academy Lab Writeup

## Objective

Enumerate the SMTP service on the target to find the banner/version and discover a valid system username.

---

## Phase 1: Banner Grabbing

**Goal:** Identify the SMTP service version.

```bash
nmap 10.129.15.250 -p25 -sC -sV
```

**Why:** `-sC` runs default scripts (including `smtp-commands`), `-sV` detects service version. This reveals the software, supported commands, and hostname.

**Result:**

```
25/tcp open  smtp    Postfix smtpd
smtp-commands: mail1, PIPELINING, SIZE 10240000, VRFY, ETRN, STARTTLS...
Service Info: Host: InFreight
```

Then confirmed via telnet:

```bash
telnet 10.129.15.250 25
```

**Result:** `220 InFreight ESMTP v2.11` ← **banner answer**

---

## Phase 2: Username Enumeration via VRFY

**Goal:** Find a valid system username using the SMTP `VRFY` command.

**What VRFY does:** Asks the server to verify if a user exists in its local recipient table. Responses:

|Code|Meaning|
|---|---|
|`250`|User exists|
|`252`|Server can't confirm but won't deny — positive hit|
|`550`|User does not exist|

**First attempt — smtp-user-enum:**

```bash
./smtp-user-enum.pl -M VRFY -U footprinting-wordlist.txt -t 10.129.15.250 -T 30
```

Returned 0 results due to high server response times.

**Second attempt — manual bash loop:**

```bash
for user in $(cat footprinting-wordlist.txt); do
    result=$(echo "VRFY $user" | nc -w 5 10.129.15.250 25 2>/dev/null)
    if echo "$result" | grep -q "252\|250"; then
        echo "[+] Valid user: $user"
    fi
done
```

**Why:** Automated tools timed out due to slow server responses. A manual bash loop with `netcat` gave more control over timing.

**Result:** `252 2.0.0 robin` ← **username answer: `robin`**

---

## Lessons Learned

1. **Always grab the banner first** — telnet to port 25 is the fastest way to confirm the service version.
2. **VRFY is a powerful enumeration vector** — if enabled, it allows username discovery without authentication.
3. **Automated tools can fail on slow servers** — when `smtp-user-enum` returns 0 results, don't trust it blindly. Verify manually first.
4. **A bash loop with netcat is a reliable fallback** — simple, controllable, and doesn't depend on external Perl dependencies.
5. **`252` is still a positive hit** — servers use it to avoid confirming users directly, but it's distinct from `550` and indicates the user likely exists.

## Command Reference

| Command                                           | Purpose                                         |
| ------------------------------------------------- | ----------------------------------------------- |
| `nmap -p25 -sC -sV <IP>`                          | Enumerate SMTP service and supported commands   |
| `telnet <IP> 25`                                  | Manual banner grab and interactive VRFY testing |
| `smtp-user-enum.pl -M VRFY -U <wordlist> -t <IP>` | Automated username enumeration                  |
| `echo "VRFY <user>" \| nc -w 5 <IP> 25`           | Manual single-user VRFY check via netcat        |
