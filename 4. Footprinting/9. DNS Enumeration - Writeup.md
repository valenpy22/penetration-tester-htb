# Objective
Enumerate a target DNS server to find all zones, records, and specifically the FQDN of a host whose last octet ends in `.203`.

## Phase 1: Initial Recon - Zone Transfer on Root Domain

## Phase 3: Brute-Force DNS - The Key Insight
- Zones that allow AXFR
- Zones that deny AXFR
- A zone must have a SOA record to be valid
- Different worldlists contain different entries
Brute-force the `dev.inlanefreight.htb` zone, which denied AXFR but is a valid zone discovered earlier.
#### Command
```
dnsenum --dnsserver 10.129.15.250 --enum -p 0 -s 0 \
-f /usr/share/seclists/Discovery/DNS/fierce-hostlist.txt \
dev.inlanefreight.htb
```

**Why `dnsenum`:** Automates brute-force DNS enumeration by querying the target server for each entry in the wordlist. Much faster than manual queries.

**Why `fierce-hostlist.txt`:** The `subdomains-top1million` wordlist didn't contain the target hostname. The hint explicitly stated that different wordlists have different entries — `fierce-hostlist.txt` is a smaller, more targeted list that happened to contain the answer.

**Answer:** `win2k.dev.inlanefreight.htb`
### Lessons Learned
1. **Always attempt AXFR first** — it's the fastest way to dump an entire zone. A server that allows it is critically misconfigured.
2. **Treat every discovered subdomain as a potential zone** — each one may have its own records and transfer policies.
3. **When AXFR fails, brute-force** — zones that restrict transfers still respond to individual A record queries.
4. **Wordlist choice is critical** — the same brute-force with a different wordlist can yield completely different results. When stuck, try multiple wordlists.
5. **SPF TXT records leak internal IPs** — always parse TXT records carefully; they often reveal internal infrastructure not exposed elsewhere.
6. **Secondary DNS servers may have different configs** — always try discovered IPs as potential DNS servers.

|Command|Purpose|
|---|---|
|`dig NS inlanefreight.htb @<IP>`|Query nameserver records|
|`dig axfr <zone> @<IP>`|Attempt zone transfer — dumps all records if allowed|
|`dnsenum --dnsserver <IP> -f <wordlist> <zone>`|Brute-force subdomains against a specific DNS server|
