## Objective

Enumerate IMAP and POP3 services on the target to extract organization info, service versions, admin email, and flags stored in mailboxes.

---

## Phase 1: Service Enumeration with Nmap

**Goal:** Identify all mail-related services, versions, and certificate information.

```bash
sudo nmap 10.129.15.250 -p110,143,993,995 -sC -sV
```

**Why:** Scanning all four mail ports at once (`110` POP3, `143` IMAP, `993` IMAPS, `995` POP3S) with `-sC` extracts SSL certificate details which leak organization info.

**Result:**

|Port|Service|Info|
|---|---|---|
|110|POP3|Dovecot pop3d|
|143|IMAP|Dovecot imapd|
|993|IMAPS (SSL)|Dovecot imapd|
|995|POP3S (SSL)|Dovecot pop3d|

From the SSL certificate:

- **Organization:** `InlaneFreight Ltd` ← answer
- **FQDN:** `dev.inlanefreight.htb` ← answer

---

## Phase 2: Connecting via SSL — Banner Grabbing

**Goal:** Connect to encrypted mail services and grab banners.

```bash
openssl s_client -connect 10.129.15.250:imaps
openssl s_client -connect 10.129.15.250:pop3s
```

**Why:** Plain telnet won't work on SSL ports (`993`, `995`). `openssl s_client` establishes the TLS handshake and drops into an interactive session, also exposing the full certificate chain with metadata.

**IMAP banner result:**

```
* OK [...] HTB{roncfbw7iszerd7shni7jr2343zhrj}
```

Flag was in the server greeting — no authentication needed. ← answer

**POP3 banner result:**

```
+OK InFreight POP3 v9.188
```

Customized POP3 version. ← answer

## Phase 3: IMAP Authentication and Mailbox Enumeration

**Goal:** Log in as `robin` and enumerate mailboxes.

After connecting via `openssl s_client -connect 10.129.15.250:imaps`:

```
a1 LOGIN robin robin
a2 LIST "" *
```

**Why:** Every IMAP command requires a tag prefix (`a1`, `a2`, etc.) so the server can match responses to commands. `LIST "" *` shows all mailboxes.

**Result:**

```
* LIST (\Noselect \HasChildren) "." DEV
* LIST (\Noselect \HasChildren) "." DEV.DEPARTMENT
* LIST (\HasNoChildren) "." DEV.DEPARTMENT.INT
* LIST (\HasNoChildren) "." INBOX
```

INBOX was empty (`0 EXISTS`). Target mailbox: `DEV.DEPARTMENT.INT`.

**Common mistake made:** Repeatedly typing `DEV.DEPARMENT.INT` (missing the `T` in `DEPARTMENT`) — always double-check mailbox names.

## Phase 4: Reading Emails — Flag Retrieval

**Goal:** Access `DEV.DEPARTMENT.INT` and read its contents.

```
a2 SELECT DEV.DEPARTMENT.INT
a3 FETCH 1:* (ENVELOPE)
a4 FETCH 1 BODY[]
```

**Why:**

- `SELECT` opens the mailbox
- `FETCH 1:* (ENVELOPE)` lists all message headers (sender, recipient, subject, date)
- `FETCH 1 BODY[]` retrieves the full email body

**Envelope result:**

```
Subject: Flag
From: CTO <devadmin@inlanefreight.htb>
To: Robin <robin@inlanefreight.htb>
```

Admin email: `devadmin@inlanefreight.htb` ← answer

**Body result:**

```
HTB{983uzn8jmfgpd8jmof8c34n7zio}
```

← answer

## Lessons Learned

1. **SSL certificates are a goldmine** — `openssl s_client` exposes organization name, FQDN, and email addresses before any authentication.
2. **Always check the server banner** — flags and version info can appear in the greeting before you even log in.
3. **IMAP requires command tags** — every command needs a prefix (`a1`, `a2`...). Forgetting this causes `BAD` errors.
4. **INBOX being empty doesn't mean no mail** — always enumerate all mailboxes with `LIST "" *` and check each one.
5. **Typos kill sessions** — `DEPARMENT` vs `DEPARTMENT` wasted multiple connection attempts. Read mailbox names carefully from `LIST` output.
6. **POP3 and IMAP serve different purposes** — POP3 is simpler (list/retrieve/delete only); IMAP supports folders and is richer for enumeration.

## Command Reference

|Command|Purpose|
|---|---|
|`nmap -p110,143,993,995 -sC -sV <IP>`|Enumerate all mail ports and extract SSL cert info|
|`openssl s_client -connect <IP>:imaps`|Connect to IMAP over SSL, grab banner and certificate|
|`openssl s_client -connect <IP>:pop3s`|Connect to POP3 over SSL|
|`a1 LOGIN user pass`|Authenticate to IMAP|
|`a2 LIST "" *`|List all mailboxes|
|`a3 SELECT <mailbox>`|Open a mailbox|
|`a4 FETCH 1:* (ENVELOPE)`|List all email headers|
|`a5 FETCH 1 BODY[]`|Read full email body|
